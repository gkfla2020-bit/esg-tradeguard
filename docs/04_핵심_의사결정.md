# 4. 핵심 기술 의사결정

## GAN vs CNN 선택

### 초기 가설
GAN(Generative Adversarial Network)으로 위성 이미지를 생성/비교하여 산림전용을 검출하려 했습니다.

### 문제점
- GAN은 이미지 "생성"에 특화 — 산림전용 "검출"과 목적이 다름
- 재현성(reproducibility)이 낮아 규제 감사에 부적합
- 생성된 이미지의 근거를 설명할 수 없음 (XAI 불가)

### 최종 결정: CNN 주력
- **U-Net (ResNet34)**: Semantic Segmentation으로 산림/농지/나지/도시 픽셀 단위 분류
- **Grad-CAM**: CNN이 "어디를 보고 판단했는지" 히트맵으로 시각화 → 설명 가능한 AI
- **NDVI 시계열**: 위성 밴드 연산으로 식생 건강도 정량화 → 수치 근거

### 선배(안지홍) 피드백
> "CNN이 더 가볍고 편합니다. 애초에 CNN에 있는 CAM을 응용한 거라서 CNN쪽으로 가면 오히려 더 논문 주장은 강해요."
> "19년도 이미지랑 26년도 이미지 차이를 보는 게 좀 더 맞는 방법이라고 봅니다."

### 판정 엔진 비율
| 방법 | 비중 | 역할 |
|------|------|------|
| CNN Segmentation + Grad-CAM | 80% | 주력 판정 |
| GIS/지수 기반 Rule (NDVI 등) | 15% | 정량적 보조 |
| GAN (향후) | 5% | 보조/미래 대비 |

## 파이프라인 순서 재구성

### 기존
서류 → OCR → **규제심사** → CBAM → 위성

### 문제
규제심사(Step 3)에서 "위성 결과 산림 31%p 감소"를 참조하는데, 위성 검증(Step 5)이 아직 안 끝남 → 논리적 모순

### 변경 후
서류 → OCR → CBAM → 위성 → **통합 보고서**

이유: 모든 검증이 끝난 후에 통합 보고서로 결과를 종합하는 것이 논리적. 위성 검증 결과가 보고서의 EUDR 판정에 자동 반영됨.

## UI/UX 디자인 원칙

1. **Monochrome**: ink(#0A0A0A) + surface(#FAFAF9) + muted 계열만. 색은 차트 데이터에만.
2. **사용자 주도**: 자동 실행 ✗ → "분석 실행" 버튼 클릭 필요
3. **경고박스 금지**: 노란/파란 배경 카드 사용 안 함 (AI 생성 느낌)
4. **절제**: 큰 것보다 작고 정확하게. 과한 애니메이션/색상 자제.
